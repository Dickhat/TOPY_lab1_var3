# Архитектура проекта

## Обзор

Проект представляет собой приложение для демонстрации методов оптимизации, реализованное на Rust с использованием фреймворка eframe/egui. Приложение поддерживает несколько лабораторных работ, каждая из которых организована в отдельной папке в `src/tabs/`.

## Структура проекта

```
src/
├── main.rs              # Точка входа в приложение
├── app/
│   ├── mod.rs           # Основная логика приложения, управление состоянием
│   └── ui.rs            # Рендеринг пользовательского интерфейса
├── tabs/
│   ├── mod.rs           # Трейт Tab и TabManager для управления вкладками
│   ├── lab1/            # Первая лабораторная работа
│   │   ├── mod.rs       # Структура Lab1Tab и основные методы
│   │   ├── models.rs    # Структуры данных (Iteration, OptimizationResult)
```

## Архитектура лабораторных работ

Каждая лабораторная работа (`lab*`) следует единой структуре для обеспечения консистентности и удобства сопровождения:

### Обязательные файлы

1. **`mod.rs`** - Основной модуль лабораторной работы
   - Определяет структуру `Lab*Tab` с полями для параметров, результатов и состояния UI
   - Реализует трейт `Tab` для интеграции с системой вкладок
   - Содержит бизнес-логику: методы для запуска оптимизации, сброса результатов и т.д.

2. **`models.rs`** - Структуры данных
   - Определяет структуры для результатов оптимизации
   - Определяет структуры для итераций и промежуточных данных
   - Может содержать вспомогательные структуры для алгоритмов

3. **`logic.rs`** - Ключевые алгоритмы
   - Содержит реализации основных алгоритмов оптимизации
   - Функции для вычисления целевых функций
   - Вспомогательные структуры (например, `Func` для подсчета вызовов)

4. **`ui.rs`** - Пользовательский интерфейс
   - Реализует методы `render_ui()` для основной панели результатов
   - Реализует методы `side_panel()` для панели параметров
   - Обрабатывает рендеринг графиков, таблиц, анимаций

5. **`export.rs`** - Экспорт результатов
   - Функции для экспорта данных в файлы
   - Поддержка различных форматов (CSV, JSON, etc.)
   - Диалоги выбора параметров экспорта

### Соглашения по именованию

- Структуры: `Lab{N}Tab` (например, `Lab1Tab`, `Lab2Tab`)
- Результаты: `{Algorithm}Result` (например, `OptimizationResult`, `HJResult`)
- Итерации: `{Algorithm}IterationData` (например, `HJIterationData`)

### Интеграция с основной системой

- Каждая лабораторная работа реализует трейт `Tab` из `src/tabs/mod.rs`
- `TabManager` в `src/tabs/mod.rs` управляет коллекцией вкладок
- Основное приложение в `src/app/` использует `TabManager` для рендеринга

## Зависимости

- **eframe/egui**: GUI фреймворк
- **egui_plot**: Для построения графиков
- **serde**: Сериализация данных
- **csv**: Экспорт в CSV

## Сборка и запуск

```bash
cargo build --release
cargo run --release
```

## Добавление новой лабораторной работы

1. Создать папку `src/tabs/lab{N}/`
2. Реализовать все обязательные файлы по шаблону существующих lab
3. Добавить модуль в `src/tabs/mod.rs`
4. Зарегистрировать в `TabManager`
