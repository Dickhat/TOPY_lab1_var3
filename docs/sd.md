# 🛠 Техническая документация (System Design)

**Проект:** Методы одномерной оптимизации (Лабораторная работа №1, Вариант 3)  
**Язык разработки:** Rust  
**Графический стек:** `eframe` (egui)  
**Технологии экспорта:** `docx-rs`, `plotters`, `image`

## 1. Архитектура проекта

Проект разделен на модули для обеспечения чистоты кода и разделения ответственности (Separation of Concerns):

*   **`main.rs`**: Точка входа. Инициализирует настройки окна и запускает цикл приложения.
*   **`logic.rs`**: "Чистая" математика. Содержит реализацию методов Дихотомии, Золотого сечения и Фибоначчи. Модуль ничего не знает об интерфейсе.
*   **`models.rs`**: Описание структур данных (`Iteration`, `OptimizationResult`), которые используются во всех частях программы.
*   **`utils.rs`**: Вспомогательные функции (линейная интерполяция `lerp`).
*   **`app/mod.rs`**: Главный управляющий модуль интерфейса. Хранит состояние приложения и диспетчеризирует отрисовку.
*   **`app/ui.rs`**: Описание внешнего вида панелей (настройки, график, таблица) и диалоговых окон.
*   **`app/export.rs`**: Логика рендеринга графиков в PNG и формирования документов Word.

## 2. Математическое обеспечение

Программа работает с двумя целевыми функциями:
1.  **F1 (Max):** $3x - x^3$ на интервале $[-3, 3]$
2.  **F2 (Min):** $(9 - x^2)/(x^2 + 2x + 3)$ на интервале $[-10, -3]$

### Методы оптимизации:
*   **Дихотомический поиск**: Использует константу различимости $\epsilon$ для разбиения интервала пополам. Требует 2 вычисления функции на итерацию.
*   **Золотое сечение**: Оптимальный метод, использующий коэффициент $0.618$. Требует 1 вычисление функции на итерацию (после первой).
*   **Метод Фибоначчи**: Использует последовательность чисел Фибоначчи для определения точек. Количество итераций $n$ вычисляется заранее исходя из точности $l$.

## 3. Ключевые технические решения

### Подсчет вычислений функции
Для реализации **Пункта 4** задания в `logic.rs` введена структура-обертка:
```rust
pub struct Func<'a> {
    pub f: &'a dyn Fn(f64) -> f64,
    pub calls: Cell<usize>, // Внутренняя изменяемость для счетчика
}
```
Это позволяет прозрачно считать количество обращений к функции без изменения логики самих алгоритмов.

### Плавная анимация (Lerp)
Для визуализации процесса сужения интервала используется линейная интерполяция координат между шагом $K$ и $K+1$. Состояние графических линий в момент времени $t \in [0, 1]$ вычисляется как:
$$ Position_{current} = Position_{start} + (Position_{end} - Position_{start}) \times t $$

### Валидация входных данных
Интерфейс блокирует возможность запуска расчетов при нарушении условий:
1.  **Интервал**: $a \le b$
2.  **Точность**: $2\epsilon < l$ (необходимо для корректной работы Дихотомии)
3.  **Положительность**: $\epsilon > 0, l > 0$

## 4. Схема экспорта

Экспорт в `.docx` реализован в двух режимах:
1.  **Краткий отчет**: Включает итоговый график, полную таблицу всех итераций и текстовое заключение (x*, f(x*), количество вызовов).
2.  **Пошаговый отчет**: Генерирует серию PNG-изображений для выбранного диапазона итераций. Каждое изображение снабжается текстовым описанием параметров шага.
